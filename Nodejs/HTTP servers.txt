Node.js me HTTP server banane ke liye hum built-in http module use karte hain.
Isse bina kisi framework (jaise Express) ke bhi server create ho sakta hai.

1️⃣ Basic HTTP Server
const http = require('http');

const server = http.createServer((req, res) => {
  res.write('Hello from Node.js server');
  res.end();
});

server.listen(3000, () => {
  console.log('Server running on port 3000');
});

Flow samjho:

createServer() → server banata hai

(req, res) → request & response objects

res.write() → data send karta hai

res.end() → response finish karta hai

listen(3000) → port par server start

2️⃣ Routing Handle karna (URL ke basis par)
const http = require('http');

const server = http.createServer((req, res) => {

  if (req.url === '/') {
    res.end('Home Page');

  } else if (req.url === '/about') {
    res.end('About Page');

  } else {
    res.statusCode = 404;
    res.end('Page Not Found');
  }
});

server.listen(3000);


➡ Isse basic routing ho jati hai.

3️⃣ JSON Response bhejna
const http = require('http');

http.createServer((req, res) => {
  res.setHeader('Content-Type', 'application/json');

  const data = { name: 'Shubham', role: 'Developer' };

  res.end(JSON.stringify(data));
}).listen(3000);

4️⃣ POST Data handle karna
const http = require('http');

http.createServer((req, res) => {

  if (req.method === 'POST') {
    let body = '';

    req.on('data', chunk => {
      body += chunk;
    });

    req.on('end', () => {
      console.log(body);
      res.end('Data received');
    });
  }

}).listen(3000);

5️⃣ Why Express is Used Instead

Node ka raw HTTP server:

Manual routing

Manual body parsing

Zyada boilerplate code

Isliye real projects me:
➡ Express.js use hota hai (fast & simple routing).

Interview Short Answer

Node.js me HTTP server http core module se banaya jata hai using
http.createServer(req, res) function.
Server event-driven, non-blocking hota hai aur multiple requests efficiently handle karta hai.
Real-world apps me usually Express.js framework use kiya jata hai for easier routing and middleware.
